# Ubuntu 系統目錄與配置筆記
## 跟目錄相關
### `/bin`
存放基本命令，如:`ls`、`cat`
### `/sbin`
存放系統命令，需要用root執行，如:`ls`、`cat`
### `/lib`
存放開機使用模組及運行使用函試
### `/etc`
存放系統配置檔案(不可共享目錄):
- `/etc/profile`: 系統環境變數配置檔 | Alias 設定
     - 內容
       ```plaintext
        export PATH="$PATH:/usr/local/jdk"
       
        alias ls='ls --color=auto'
       ``` 
     - 修改:
     ````bash
     source /etc/profile
     ````
     - 參考:
       https://ithelp.ithome.com.tw/m/articles/10328121    
- `/netplan/*.yaml`:網路設定配置 (需安裝netplan.io)
-      - 內容:
          ```plaintext
          network:
            version: 2
            renderer: networkd
            ethernets:
              eth1:
                addresses:
                  - 192.168.10.2/24
                nameservers:
                  search: [liho.tw, otherdomain]
                  addresses: [8.8.8.8, 8.8.4.4]
                routes:
                  - to: default
                    via: 192.168.10.1
          ```
       修改:
        ```
        netplan try
        ```
        參考:
        https://zonego.tw/post/ubuntu-netplan/      
- `/etc/systemd/resolv.conf`: DNS 伺服器配置
     - 內容:
       ```plaintext
       DNS = 8.8.8.8 8.8.4.4
       Domaina = twnic.com.tw  
       ```
       修改:
       ```bash
       systemctl restart systemd-resolved
       systemd-resolve --status
       ```

- `etc/systemd/system/XXX_service.service` : 自訂`systemd`服務設定檔
     - 內容:
       ```plaintext
       [Unit]
            # 服務名稱
            Description=Your Server
                    
            # 服務相關文件
            # Documentation=https://example.com
            # Documentation=man:nginx(8)
                    
            # 設定服務啟動的先後相依姓，例如在網路啟動之後：
            # After=network.target
               
       [Service]
            # 行程類型
            Type=simple     
                    
            # 啟動服務指令
            xecStart=/opt/XXX.py
                    
            # 服務行程 PID（通常配合 forking 的服務使用）
            # PIDFile=/run/your_server.pid
               
            # 啟動服務前，執行的指令
            # ExecStartPre=/opt/your_command
                         
            # 啟動服務後，執行的指令
            # ExecStartPost=/opt/your_command
                         
            # 停止服務指令
            # ExecStop=/opt/your_command
                         
            # 停止服務後，執行的指令
            # ExecStopPost=/opt/your_command
                         
            # 重新載入服務指令
            # ExecReload=/opt/your_command
                         
            # 服務終止時自動重新啟動
            Restart=always
                         
            # 重新啟動時間格時間（預設為 100ms）
            # RestartSec=3s
                         
            # 啟動服務逾時秒數
            # TimeoutStartSec=3s
                         
            # 停止服務逾時秒數
            # TimeoutStopSec=3s
                         
            # 執行時的工作目錄
            # WorkingDirectory=/opt/your_folder
                         
            # 執行服務的使用者（名稱或 ID 皆可）
            # User=myuser
                         
            # 執行服務的群組（名稱或 ID 皆可）
            # User=mygroup
                         
            # 環境變數設定
            # Environment="VAR1=word1 word2" VAR2=word3 "VAR3=$word 5 6"
                         
            # 服務輸出訊息導向設定
            # StandardOutput=syslog
                         
            # 服務錯誤訊息導向設定
            # StandardError=syslog
                         
            # 設定服務在 Syslog 中的名稱
            # SyslogIdentifier=your-server 
       [Install]
            #啟動目標
            WantedBy=multi-user.target
          ```
     修改:
       ```bash
          # 權限設定
          sudo chmod 644 /etc/systemd/system/XXX_service.service
          # 重新載入 Systemd 設定檔
          sudo systemctl daemon-reload
          # 啟動自訂的服務
          sudo systemctl start XXX_service
          # 查看服務狀態
          systemctl status XXX_service
          # 停止服務
          sudo systemctl stop XXX_service
          # 設定開機自動啟動服務
          # sudo systemctl enable XXX_service
          # 取消開機自動啟動服務
          sudo systemctl disable XXX_service
       ```
  - `etc/apt/sources.list.d/` : apt下載位置及配置
     
- `/ssh/sshd_config/` => SSH配置檔 (需安裝ssh)
     - 內容:
          ```plaintext
               #連接portt
               Port 22
               #root登入
               PermitRotLogin yes
               #限定登入者
               AllowUsers <username>
           ```
     修改:
       ```
       sudo systemctl restart ssh
       ```
  
- `etc/pam.d`: PAM身分驗證模組
- `etc/passwd`: 帳號權限
- `etc/shadow`: 帳號加密密碼及有效時間
- `etc/group`: 群組
- `etc/fstab`: 設備掛載資訊
- `etc/hostname`: 主機名稱
- `etc/hosts`: LAN　ip預名
- `etc/motd`: 登入歡迎訊息
- `etc/sudoers `: sudo 配置檔 (使用uisudo編輯)
        - 內容:
        ```plaintext
             %group_name  ALL=(ALL) ALL  #群組設定
             user_name  ALL=(ALL) ALL  #用戶設定
             user_name  ALL=(ALL) NOPASSWD:ALL #不須密碼

             #自訂規則
             User_Alias MYACC = user_a, user_b, user_c
             MYACC  ALL=(root) !/usr/bin/passwd,
             /usr/bin/passwd [A-Za-z]*, !/usr/bin/passwd  root 
             
        ```
        
- `/bash.bashrc` :同profile
- `/sysctl.conf` :網路進階配置
   - 修改:
     ```bash
        -sysctl -p 
     ```   
- `/services` : 運行中網路服務端口及協定
- `/opt` : 存放不可共享的第三方軟體

## 用戶相關
### `/home/<username>` : 個別用戶的檔案及數據
- ~/.bashrc :設定使用者環境變數
 - 內容:
   ```plaintext
    export JAVA_HOME=~/java/jdk-<version>
    export PATH=$JAVA_HOME/bin:$PATH 
   ``` 
   修改:
   ```bash
   source ~/.bashrc
   ```
## 系統功能目錄
### `/tmp` : 暫存檔案(需手動或用腳本刪除)
### `/var` : 常態變動檔案 (log,mail,db_data)
### `/proc` : 進程檔案
### `/sys` : 硬體資訊
### `/dev` : 硬碟
- `/dev/null` : 丟棄不必要的輸出
   ```bash
    ... > /dev/null
   ```     
### `/boot` : 開幾檔案
### `/srv` : 各類服務資料
### `/mnt` : 掛載點
### `/media` : 可移除設備 USB
### `/opt` : 不共享第三方軟體及腳本


## 第三方及共享目錄
### `/usr` : 下載後不會修改的檔案
- `/usr/bin` : 基本命令(ls,cat)、預設軟體(vim,python) 
- `/usr/sbin` : 非系統正常運作所需命令(systemctl,ifconfig、守護進程服務(nginx,apache)
- `/usr/local` : 第三方軟體(jdk,mvn,npm)
- `/usr/share` : 線上說明文件
   -`/usr/share/keyrings` : 密鑰
